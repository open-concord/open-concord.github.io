This document pertains to the structure of Concord implementations. For implementation agnostic standards, see std

We provide a set of libraries to expedite the process of creation:

  • Core, the core tree and graph abstraction
  • Uttu, a p2p networking library
  • Tamuz, an HME implementation.

These libraries provide base functionality, and allow you to participate in concord. In order to actually interpret and produce meaningful information on concord, you need to use interpretive protocols.

These range from networking protocols, to exchange protocols, to more arbitrary protocols, such as messaging, file sharing, voice calls, ectera. OpenConcord maintains a subset of these protocols, which can be found in proto.

Generally, implementions will be organised in such a fashion: Networking -> (Storage <-> PoW) -> Content Protocol. It's that simple!
